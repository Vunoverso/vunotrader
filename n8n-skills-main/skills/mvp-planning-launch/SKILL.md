# Planejamento de MVP & Lançamento

Como sair da ideia para o produto no ar no menor tempo possível.

## O Conceito de MVP (Minimum Viable Product)

### 1. Definição do Escopo Core
- O que é o "problema real" que seu app resolve? Elimine tudo o que não é essencial para essa solução.
- Use a regra dos **80/20**: 20% das funcionalidades resolvem 80% do problema do usuário.

### 2. Priorização Ágil
- Utilize o método **MoSCoW** (Must have, Should have, Could have, Won't have).

### 3. Ciclo de Feedback
- Lance rápido (Build -> Measure -> Learn). É melhor um produto simples no ar do que um perfeito na gaveta.

## Preparação para o Lançamento
- **Landing Page**: Crie uma página de captura antes mesmo do app estar pronto.
- **Soft Launch**: Libere para um grupo pequeno de usuários primeiro.
- **Monitoring**: Tenha ferramentas de análise ligadas desde o primeiro dia.
