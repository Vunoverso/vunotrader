# Pasta Evolucao

## Finalidade

Esta pasta guarda o historico vivo do projeto.

Tudo que for importante para arquitetura, melhoria, correcao, decisao, risco, ajuste de rota ou novo modulo deve ser registrado aqui.

## Regra de uso

Antes de aplicar codigo novo, revisar esta pasta.

Depois de aplicar mudanca relevante, registrar a evolucao.

## Quando criar novo arquivo

Criar um arquivo novo quando houver:

- nova ideia de produto
- novo modulo
- nova integracao
- nova estrategia tecnica
- correcao relevante
- revisao de arquitetura
- mudanca importante de fluxo

## Quando atualizar arquivo existente

Atualizar o mesmo arquivo quando a mudanca for continuacao direta do mesmo assunto.

## Estrutura recomendada

Cada arquivo deve conter, sempre que possivel:

- contexto
- problema ou objetivo
- decisao
- impacto tecnico
- arquivos afetados
- riscos
- proximos passos

## Convencao de nomes

- AAAA-MM-DD-tema.md
- AAAA-MM-DD-modulo-ajuste.md
- AAAA-MM-DD-correcao-nome.md

## Regra anti retrabalho

Se um arquivo desta pasta mostrar que algo ja foi feito, pensado, descartado ou corrigido, isso deve ser respeitado antes de qualquer nova implementacao.

Se o codigo divergir da evolucao registrada, a divergencia deve ser sinalizada.
